## Purpose

OS-APOW is a headless agentic orchestration platform that transforms GitHub Issues into automated Execution Orders. It dispatches specialized AI agents (opencode workers) in isolated Docker DevContainers to autonomously clone repositories, generate code, run tests, and submit Pull Requests.

## Tech Stack

- **Language:** Python 3.12+
- **Framework:** FastAPI 0.110+ (async web API)
- **Package Manager:** uv (Rust-based, deterministic builds)
- **Database:** PostgreSQL 16 (via docker-compose)
- **Cache/Queue:** Redis 7 (via docker-compose)
- **State Management:** GitHub Issues + Labels ("Markdown as a Database")
- **Containerization:** Docker + Docker Compose
- **CI/CD:** GitHub Actions (all actions pinned to commit SHAs)
- **AI Agent Runtime:** opencode CLI with GLM-5 (ZhipuAI)
- **Linting:** Ruff (replaces Flake8, isort, Black)
- **Type Checking:** mypy (strict mode)
- **Testing:** pytest + pytest-asyncio + respx

## Key Architectural Decisions

1. **ADR-01: Standardized Shell-Bridge Execution** — Sentinel interacts with workers exclusively via `./scripts/devcontainer-opencode.sh`
2. **ADR-02: Polling-First Resiliency** — Polling as primary discovery; webhooks are an optimization
3. **ADR-03: Provider-Agnostic Interface** — `ITaskQueue` abstraction enables future backends (Linear, Notion)
4. **ADR-04: Markdown as a Database** — GitHub Issues + Labels as state store for transparency
5. **ADR-05: Logic-as-Markdown** — AI behavior defined in markdown modules, not hardcoded
