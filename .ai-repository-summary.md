## Architecture Overview — Four Pillars

### 1. The Ear — Work Event Notifier (`notifier_service.py`)
- **Technology**: FastAPI + Uvicorn + Pydantic
- **Port**: 8000
- **Endpoints**: `POST /webhooks/github`, `GET /health`, `GET /docs`
- **Security**: HMAC SHA256 signature validation on all webhook requests

### 2. The State — Work Queue (GitHub Issues)
- **Philosophy**: "Markdown-as-a-Database"
- **State Machine**: Labels drive transitions (`agent:queued` → `agent:in-progress` → `agent:success`/`agent:error`)
- **Concurrency**: GitHub Assignees serve as distributed locks

### 3. The Brain — Sentinel Orchestrator (`orchestrator_sentinel.py`)
- **Technology**: Python asyncio, HTTPX, subprocess calls to shell bridge
- **Behavior**: Polls for `agent:queued` issues every 60s, claims tasks, dispatches workers
- **Resilience**: Jittered backoff, reconciliation loop for stale tasks, cost guardrails

### 4. The Hands — Opencode Worker
- **Technology**: Docker DevContainer + opencode CLI + LLM
- **Isolation**: Segregated Docker bridge network, 2 CPU / 4GB RAM limits
- **Execution**: Reads instruction modules, runs tests, creates PRs

## Data Flow (Happy Path)

1. **Stimulus** — User opens GitHub Issue with specification template
2. **Notify** — GitHub Webhook hits The Ear (FastAPI)
3. **Triage** — Notifier verifies HMAC, applies `agent:queued` label
4. **Claim** — Sentinel detects queued issue, assigns itself, updates to `agent:in-progress`
5. **Execute** — Sentinel provisions DevContainer, dispatches AI worker
6. **Finalize** — Worker completes, Sentinel applies terminal label, creates PR

## Key Design Decisions

| ADR | Decision | Rationale |
|-----|----------|-----------|
| ADR-01 | Shell-bridge execution | Keep complex Docker logic in scripts; Python handles state |
| ADR-02 | Polling-first resiliency | Self-healing after downtime; webhooks are optimization layer |
| ADR-03 | Provider-agnostic ITaskQueue | Strategy pattern prevents vendor lock-in |
| ADR-04 | Markdown-as-a-Database | GitHub Issues provide audit, versioning, and human intervention |
| ADR-05 | Logic-as-Markdown | AI behavior updated via PR without redeploying Python |
